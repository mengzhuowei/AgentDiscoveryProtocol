# 贡献指南

**更新：** 2026-05-15

---

感谢你对 ADP 协议的兴趣！欢迎通过以下方式参与。

## 贡献方式

### 1. 反馈与讨论

- **提 Issue**：协议规范有歧义、遗漏或错误
- **讨论**：用例场景、扩展建议、实现经验
- **任何疑问**：文档看不懂、术语不一致、示例不够清晰

### 2. 文档贡献

协议规范文档位于 `adp-spec/` 目录，均为 Markdown 格式。

提交文档修改时注意：

- 保持文档风格一致（标题层级、表格格式、代码块语言标注）
- 新增术语请在 `README.md` 的术语表中同步补充
- 修改核心协议内容（Envelope 格式、API 路径、能力定义）需标注版本变更

### 3. 参考实现

我们欢迎各语言的参考实现：

| 语言 | 组件 |
|---|---|
| TypeScript | SDK / Registry / Relay |
| Python | SDK |
| Go | Registry / Relay |
| Rust | Registry / Relay |

参考实现应严格遵循协议规范，并通过兼容性测试（待定义）。

### 4. 用例与场景

如果你用 ADP 连接了多个 Agent，欢迎分享你的用例。这有助于协议演进。

---

## 规范修改流程

1. 开 Issue 描述修改内容与理由
2. 讨论达成共识后，提交 PR 修改对应文档
3. 修改文档头部的 `更新` 日期
4. 如果修改涉及协议行为，在 `ROADMAP.md` 中记录版本变更

---

## 行为准则

- 尊重不同意见：协议设计没有完美的唯一解
- 关注场景而非偏好：讨论时应基于实际用例而非个人技术偏好
- 保持文档的质量：每条规范都应当可以被独立实现
